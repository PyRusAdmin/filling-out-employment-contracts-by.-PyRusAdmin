## Формирование дополнительных соглашений (перевод на другую работу).

1. Заполнить данные в списочный состав `data/list_gup/Списочный_состав.xlsx`.
2. Очистить базу данных `data/contracts.db`, нажатием кнопки "Очистка данных".
3. Записать табельный номер работника, которого нужно перевести на другую работу.
4. Записать данные из файла `data/list_gup/Списочный_состав.xlsx` в базу данных `data/contracts.db`, нажатием кнопки "
   Управление БД".
    * Записать начальную строку, для парсинга и записи в базу данных `data/contracts.db`.
    * Записать конечную строку для парсинга и записи в базу данных `data/contracts.db`.
5. Заполнить дополнительные договора, нажатием кнопки "Получить договор (перевод на другую работу)".

## Формирование уведомлений на сокращение

Структура: `templates/notification_compression.html`

1. Заполнить данные в списочный состав `data/list_gup/Списочный_состав.xlsx`.
2. Очистить базу данных `data/contracts.db`, нажатием кнопки "Очистка данных".
3. Записать данные со списочного состава в базу данных.
    - Ввести начальную строку для парсинга
    - Ввести конечную строку для парсинга
    - Нажать кнопку "Записать данные"
      После записи данных, программа запишет данные в базу данных `data/contracts.db`
4. Перейти во вкладку формирование договоров
5. Нажать на `Формирование уведомления на сокращение`
6. Нажать на `Сверка уведомлений`, для удаления лишних файлов

В файле `data/data.json` — список работников, которые не попадают под сокращение. Программа удаляет все файлы, кроме
тех, которые указаны в `data.json`.

Готовые уведомления на сокращение находятся в папке `output/Готовые_уведомления_сокращение`
